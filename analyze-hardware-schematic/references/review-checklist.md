# Review Checklist

Use this checklist before finalizing output.

## Evidence Integrity

- Every important claim can be traced to a screenshot, BOM row, or explicitly labeled inference.
- Topology claims are not derived from BOM names alone.
- Cross-module claims are only made when the relationship is actually evidenced.

## Structure

- Each module document is organized by submodule.
- Each submodule starts with key parts.
- Part identity and circuit analysis are not mixed together randomly.
- Important relations are captured in a structured table.

## Uncertainty Handling

- Unresolved items are explicit.
- Conflicts are explicit.
- Inference is labeled as `推断`, not written as fact.
- The document does not hide uncertainty for readability.

## Output Completeness

- Module-level documents exist where analysis was possible.
- The inter-module relationship document only summarizes supported links.
- The project-level open issues list captures conflicts, unresolved links, missing evidence, and user action items.

## Input Quality Handling

- Poor screenshots triggered an input-quality issue list before conclusions.
- Missing recommended inputs are noted when they reduce confidence.
